# QA 피드백 가이드

이 문서는 `우취될까?` MVP를 지인 3~5명에게 공유하고, 실제 사용자의 반응을 일관된 방식으로 수집하기 위한 기준이다.

## 목표

- 사용자가 첫 화면에서 서비스의 목적을 바로 이해하는지 확인한다.
- 날짜 선택, 경기 선택, 응원팀 설정 흐름이 자연스러운지 확인한다.
- 직관 판단 카드의 문구·점수·상세 정보가 의사결정에 도움이 되는지 확인한다.
- 모바일 환경에서 레이아웃 흔들림, 버튼 조작, 읽기 흐름의 불편함을 찾는다.
- 추가 기능 요청을 MVP 이후 과제로 분류한다.

## 권장 수집 방식

Google Forms 사용을 권장한다.

지금처럼 소규모 QA만 진행할 때는 카카오톡이나 DM으로 의견을 받아도 충분하지만, Google Forms를 쓰면 다음 장점이 있다.

- 모든 사용자가 같은 질문에 답하므로 비교가 쉽다.
- 선택형 답변과 주관식 답변을 함께 받을 수 있다.
- 응답이 자동으로 시트에 쌓여 나중에 분류하기 쉽다.
- QA 인원이 늘어나도 같은 링크를 재사용할 수 있다.
- 개선 전후의 응답을 비교하기 쉽다.

단, 지금 단계에서는 로그인을 요구하지 않고 익명 응답을 받는 편이 좋다. 이름이나 연락처는 필수로 받지 않는다.

## 지인에게 보낼 안내 문구

```text
야구 직관 가기 전에 우천취소 가능성을 가볍게 확인하는 서비스를 만들고 있어요.
아래 링크에서 오늘/내일/모레 경기 중 하나를 골라보고, 2~3분 정도만 피드백 부탁드려요.

서비스 링크:
https://build-the-dom.vercel.app

중점적으로 보고 싶은 부분은 이거예요.
1. 첫 화면에서 무슨 서비스인지 바로 이해되는지
2. 날짜와 경기를 고르는 과정이 자연스러운지
3. 카드의 판단 문구와 점수가 실제로 도움이 되는지
4. 모바일에서 불편하거나 어색한 부분이 있는지

좋았던 점도, 헷갈렸던 점도 편하게 적어주시면 됩니다!
```

## 관찰 체크리스트

사용자 응답과 별개로, 직접 옆에서 보거나 피드백을 들을 때 아래 항목을 체크한다.

- 첫 진입 후 사용자가 바로 날짜 영역을 인지했는가?
- 경기 탭이 선택 가능한 요소로 보였는가?
- 날짜 변경 후 카드 내용이 바뀐 것을 인지했는가?
- 응원팀 설정이 너무 작거나 숨겨져 있지 않았는가?
- 카드의 접힘/펼침 버튼을 발견했는가?
- 점수, 헤드라인, 상세 표 중 어떤 요소를 가장 먼저 봤는가?
- “돔구장”, “우천취소”, “경기 결과” 카드가 다른 카드와 자연스럽게 구분됐는가?
- KBO 공식 공지 링크가 신뢰 보조 장치로 느껴졌는가?
- 모바일에서 버튼 터치, 스크롤, 글자 크기가 불편하지 않았는가?

## 피드백 분류 기준

| 분류 | 기준 | 처리 |
| --- | --- | --- |
| 바로 반영 | 오타, 깨진 레이아웃, 명확한 버그, 혼란스러운 버튼 | 즉시 수정 후보 |
| 논의 후 반영 | 문구 톤, 점수 기준, 상세 정보 표현, 카드 우선순위 | 기획 논의 후 수정 |
| 보류 | 회원 기능, 알림, 예매 링크, 팀 로고 등 MVP 밖 기능 | 후속 마일스톤 후보 |
| 실험 후보 | 기본 펼침 여부, 응원팀 자동 포커스 방식, 위험도 문구 변형 | A/B 또는 소규모 재검증 |

## 완료 기준

- 최소 5명의 응답을 수집한다.
- 같은 문제가 2명 이상에게 반복되는지 확인한다.
- 바로 반영 항목과 보류 항목을 분리한다.
- MVP 공개 전에 반드시 수정할 항목을 3개 이하로 좁힌다.
