## Git 커밋 규칙

- 기능 구현 또는 트러블슈팅이 독립된 의미 단위로 끝날 때마다 커밋한다.
- 커밋 메시지는 항상 한 줄로 간결하게 작성한다.
- 접두사는 `feat:`, `fix:`, `refactor:`, `docs:`, `style:`, `test:`, `chore:` 중 하나를 사용한다.
- 접두사를 제외한 제목은 반드시 자연스러운 한국어로 작성한다.
- `수정`, `변경`, `작업`처럼 의미가 모호한 제목은 사용하지 않는다.
